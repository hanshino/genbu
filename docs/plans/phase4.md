## 2. 160 Dungeon — 迷霧九宮格 (`/tools/160`)

### 2.1 遊戲規則

- 3×3 九宮格，9 個房間各有名稱：魁、晶、阜、寶、帝、彤、牡、蒼、岡
- 每個房間需放置 1~9 的水晶數
- 每行、每列、每對角線的總和必須相同（12 或 15）
- 玩家已知 2 個房間的值，需推算其餘 7 個

### 2.2 九宮格位置對應

```
魁(0) │ 晶(1) │ 阜(2)
──────┼───────┼──────
寶(3) │ 帝(4) │ 彤(5)
──────┼───────┼──────
牡(6) │ 蒼(7) │ 岡(8)
```

### 2.3 Algorithm

移植自 `questController.js` 的 `calculate()`:

1. **輸入驗證**:
   - sum 只接受 12 或 15
   - 水晶數必須在 1~9 範圍內
   - **對角線限制**: 若兩個已知房間的位置索引加總 = 8（如 魁(0)+岡(8)、晶(1)+蒼(7)、阜(2)+牡(6)），此組合無法計算，需顯示錯誤提示「這組無法計算出結果」
   - **全等特例**: 若兩個已知格的值都等於 sum/3，則所有格都填 sum/3
2. 中間格 (帝) = sum / 3
3. 填入已知的 2 個房間
4. 迭代計算：對每個未知格，檢查其所有關聯線（行/列/對角），若同線另外兩格已知，則計算出此格的值（`value = sum - 已知1 - 已知2`）
5. 重複直到所有格都填滿（最多 5 次迭代）

## 3. 175 Dungeon — 北斗七星 (`/tools/175`)

### 3.1 遊戲規則

- 7 個開關，每個開關有「開」或「關」兩種狀態
- 遊戲給一個數字 (1-127)，玩家需要設定正確的開關組合
- 數字轉為 7 位二進位: 1=開, 0=關

### 3.2 Algorithm

```typescript
function solveSevenStar(num: number): ("開" | "關")[] {
  const binary = num.toString(2).padStart(7, "0");
  return binary.split("").map((b) => (b === "1" ? "開" : "關"));
}
```

**顯示順序注意**: LINE Bot 原始碼在顯示時對結果做了 `.reverse()`（`questController.js:169`），意味著星 1 對應最低位元（LSB），星 7 對應最高位元（MSB）。網站 UI 應保持此順序：從左到右為星 1~7，對應二進位從低位到高位（即 `result.reverse()` 後的順序）。

- 可選：也能反向操作 — 點擊星星切換開/關，自動計算對應數字

## 4. 180 Dungeon — 神武禁地 (`/tools/180`)

### 4.1 遊戲規則

- 3×3 九宮格，需填入 1~9（不重複）
- 每行、每列、每對角線總和相同 = `sum`
- 有一個數字被「封印」（leak），不能使用
- 玩家需用剩餘 8 個數字 + 1 個替代來填滿九宮格

### 4.2 Algorithm

移植自 `questController.js` 的 `getGodQuestResult()`:

1. 計算所有可能的三元組：`getAllPossible(sum * 3 - 45, leak)`
   - sum×3 - 45 是因為 1~9 總和 = 45
2. 對每個三元組嘗試排列
3. 驗證所有行/列/對角線是否滿足 sum
4. **失敗判定**: 當 `results.length <= 1` 時視為計算失敗（匹配原始 LINE Bot 邏輯 `questController.js:285`）。單一結果可能是不完整的退化解，不應顯示
5. 成功時回傳 `results[0]`（第一組有效的完整 9 格解）

```typescript
// god-quest.ts
export function solveGodQuest(sum: number, leak: number): number[] | null {
  const results = getGodQuestResult(sum, leak);
  // results.length <= 1 是失敗 — 匹配 LINE Bot 原始邏輯
  if (results.length <= 1) return null;
  return results[0]; // 9 元素陣列，對應九宮格位置
}
```

## 8. Mobile Responsive Design

三個工具都需要良好的手機體驗（玩家常在遊戲中用手機查詢）：

- 九宮格: 格子大小 `min(80px, 25vw)`，確保手機上不會太小也不會超出
- 七星: 7 個圓形橫排，小螢幕時自動換行或縮小
- 輸入框: 使用 `type="number"` + `inputmode="numeric"` 叫出數字鍵盤
- 所有按鈕/格子足夠大 (min 44px touch target)
