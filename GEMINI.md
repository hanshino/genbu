# Gemini Project Context: Genbu (玄武)

Genbu is an unofficial, open-source database and utility site for the game "TTHOL" (武林同萌傳). It provides players with tools for item lookup, equipment ranking comparison, skill browsing, and puzzle-solving.

## Project Overview

- **Purpose:** A comprehensive game database providing item search, attribute comparison, and ranking tools.
- **Main Technologies:**
    - **Frontend:** Next.js 16 (App Router), React 19, Tailwind CSS 4, Shadcn/UI, Recharts.
    - **Backend:** Node.js (Next.js Server Components), SQLite (`better-sqlite3`).
    - **Database:** `tthol.sqlite` (read-only game data).
    - **Testing:** Vitest, React Testing Library, JSDOM.
    - **Deployment:** Docker (standalone), GitHub Actions.

## Architecture & Design Patterns

- **Database Access:** A singleton connection pattern is used in `src/lib/db.ts` using `globalThis` to prevent multiple connections during HMR (Hot Module Replacement). The database is strictly read-only in production.
- **Scoring System:** Located in `src/lib/scoring/`, this module contains complex logic for calculating equipment "scores" based on weighted attributes for different player "presets" (e.g., Pure Strength, Doctor/Poison).
- **Component Organization:**
    - `src/components/ui`: Primitive UI components (mostly generated by Shadcn/UI).
    - `src/components/[domain]`: Domain-specific components (e.g., `items`, `monsters`, `ranking`).
- **Data Fetching:** Primarily uses Next.js Server Components for initial data loading via `src/lib/queries/`.

## Key Commands

- `npm run dev`: Starts the development server.
- `npm run build`: Builds the application for production.
- `npm run start`: Starts the production server.
- `npm run lint`: Runs ESLint for code quality checks.
- `npm run format`: Formats code using Prettier.
- `npm run typecheck`: Performs TypeScript type checking.
- `npm run test`: Runs the test suite using Vitest.
- `npm run sync:images`: Fetches equipment images from external sources (required for local development).
- `npm run deploy`: Deployment script (invoked with `.env` file).

## Development Conventions

- **State Management:** Prefers React's built-in state management and URL-based state (via search params) for filtering and pagination.
- **Styling:** Uses Tailwind CSS 4 with a mobile-first approach. Adheres to Shadcn/UI patterns.
- **Types:** Strictly typed with TypeScript. Shared types are located in `src/lib/types/`.
- **Testing:** All new logic (especially in `src/lib/scoring/` and `src/lib/queries/`) should be accompanied by Vitest unit tests.
- **Documentation:** Phase-based project roadmaps are maintained in `docs/plans/`.

## Important Directories

- `src/app`: Application routes and pages.
- `src/lib/scoring`: Core equipment ranking logic.
- `src/lib/queries`: Database query abstractions.
- `src/lib/constants`: i18n, item types, and game-specific constants.
- `docs/`: Technical specifications and development plans.
- `scripts/`: Utility scripts for data syncing and deployment.
